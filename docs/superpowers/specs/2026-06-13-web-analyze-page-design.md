# KB教练 — Web 拍照分析页设计文档

**日期：** 2026-06-13
**状态：** 已确认

---

## 目标

构建 KB教练 的 Web 端拍照分析 MVP 页面，让用户可以上传体态照片，获取 AI 体态分析结果。

## 范围

- 单页面应用（拍照分析页）
- 前端静态展示，后端 API 后续对接
- 不包含：用户认证、历史记录、数据持久化

---

## 页面布局

**左右分栏结构：**

| 左侧（40%） | 右侧（60%） |
|-------------|-------------|
| 照片上传区 | 分析结果面板 |

- 左侧固定宽度，右侧自适应
- 响应式：小屏（< 768px）时切换为上下布局

## 用户流程

```
上传照片 → 加载动画 → 一次性展示结果
```

1. 用户点击或拖拽上传照片
2. 显示加载状态（分析中...）
3. 分析完成后，右侧面板展示结果

## 组件拆分

## 数据模型（Mock）

```typescript
interface AnalysisResult {
  score: number;           // 体态评分 0-100
  issues: Issue[];         // 核心问题
  radar: RadarData;        // 雷达图数据
  suggestions: Suggestion[]; // 建议方案
}

interface Issue {
  name: string;            // 问题名称
  severity: 'mild' | 'moderate' | 'severe';
}

interface RadarData {
  headForward: number;     // 头前伸 0-100
  roundShoulder: number;   // 圆肩
  pelvicTilt: number;      // 骨盆前倾
  kneeExtension: number;   // 膝超伸
}

interface Suggestion {
  exercise: string;        // 动作名称
  sets: string;            // 组数次数
  description: string;     // 简要说明
}
```

## Mock 数据

MVP 阶段使用静态 Mock 数据，上传后延迟 1.5 秒返回模拟结果。

---

## 验收标准

1. 页面在 1280px+ 桌面宽度下正确显示左右分栏
2. 点击/拖拽上传照片后显示加载状态
3. 1.5 秒后展示分析结果
4. 评分和问题标签始终可见
5. 点击"查看详细分析"展开雷达图和建议
6. 再次点击可折叠
7. 移动端（< 768px）自动切换为上下布局

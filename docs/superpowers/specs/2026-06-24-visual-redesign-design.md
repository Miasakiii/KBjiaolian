# KB 教练 · 新视觉设计系统

> 状态：已落地 · 跨端验证通过（miniprogram 两轮 + Flutter 全量）
> 日期：2026-06-24
> 范围：跨端设计系统规范 + 小程序端参照落地（4 高频屏）
> 决策来源：brainstorming 视觉伴侣会话（`.superpowers/brainstorm/1404-1782312952/`）

## 1. 背景与目标

KB 教练三端（miniprogram / Flutter / Web）现状：功能完整，但 UI 高度"模板化"——草绿白底清新风是千篇一律的小程序 / Flutter 教程长相，配不上 ¥168/年 Pro 定位。三端三套图标、token 系统半摆设（miniprogram 页面硬编码颜色绕过 CSS 变量）、Flutter 出现"两种绿打架"（`#22c55e` 与 `Colors.green.shadeXXX`）、无任何数据可视化（miniprogram 8 维度只有横条；Flutter 有雷达图但彩虹图例是错的）。

**目标**：建立一套有辨识度的视觉系统（"立人设"），从"绿白清新模板"转向"临床专业 + 高端极简"。第一版在小程序端落地 4 高频屏作为参照样板，规范跨端通用。

## 2. 设计决策汇总

| 维度 | 决策 |
|------|------|
| 气质 | A 配色（深 teal + 冷灰 + 浅底，临床专业）+ C 留白（大呼吸、单点强调，高端极简） |
| 主色 | 草绿 `#22c55e` → 深 teal `#0f766e` |
| 招牌数据图 | 雷达图 2+3：result 模式（色块填充 + 问题点高亮）用于结果页/首页摘要；compare 模式（双层 before/after）用于对比页 |
| 落地端 | 小程序优先（参照样板），规范跨端写 |
| 范围 | 三件套（token + 组件 + 雷达）+ 4 高频屏（首页/分析结果/历史对比/登录） |
| 暗色模式 | 先只做 light；token 结构预留 dark 接口（占位空变量），不适配 |
| 字体 | 系统字 + 字号阶梯，不加载自定义字体；字重只用 400/600 |
| 实现架构 | 混合：简单排版用 token+工具类（真正用起变量），复杂/高复用块抽成自定义组件 |
| 图标系统 | 统一一套线性 SVG（1.8 描边/圆角/24×24），三端共用；含 tab 图标 + 8 维体态图标 + 状态语义图标 + 8 动作模式图标 + 肌群数据色板 |
| 肌群色标 | 方案 B：6 色低饱和数据色板，仅肌群分类用，不作装饰 |

## 3. 设计 Token

### 3.1 配色（light）

主色从饱和草绿换成深 teal 墨绿——更冷静、更临床可信、离模板感最远。

| Token | 值 | 用途 |
|------|------|------|
| `--brand` | `#0f766e` | 主色：按钮/链接/数据描边/强调 |
| `--brand-600` | `#0d5a54` | 按压态 |
| `--brand-soft` | `#ccfbf1` | 弱底色/标签底/图标容器底 |
| `--accent-warn` | `#f97316` | **唯一警示色**：问题点/不达标，绝不作装饰 |
| `--bg` | `#f6f8f7` | 页面底（偏冷灰白，非纯白） |
| `--surface` | `#ffffff` | 卡片 |
| `--surface-2` | `#f9fafb` | 次级面/输入框底 |
| `--text-1` | `#0f172a` | 主文字 |
| `--text-2` | `#475569` | 次文字 |
| `--text-3` | `#94a3b8` | 辅助/占位 |
| `--line` | `#e2e8e4` | 分割线/卡边（冷灰） |
| `--line-soft` | `#eef2f1` | 极弱分割/雷达网格线 |

**铁律**：`--accent-warn` 是唯一非品牌色，只用于"问题/不达标"。全屏 teal + 灰，唯一一处橙抓眼 = C 留白的"单点强调"。

**肌群数据色板**（方案 B，仅肌群分类用）：

| Token | 值 | 肌群 |
|------|------|------|
| `--data-chest` | `#0f766e` | 胸（复用 brand） |
| `--data-back` | `#4f6d7a` | 背 |
| `--data-leg` | `#6b7280` | 腿 |
| `--data-glute` | `#9f6f8f` | 臀 |
| `--data-core` | `#5b7a8c` | 核心 |
| `--data-shoulder` | `#b08968` | 肩 |

数据色板独立于品牌色体系，只出现在肌群色标点/标签底，不进入按钮、卡片、文字等通用组件。

**dark 预留**：token 结构里预留 `--bg-dark` / `--surface-dark` / `--text-1-dark` 等占位空变量，dark 适配时只填值不改结构、不改组件代码。

### 3.2 字号阶梯（系统字）

C 留白的高级感不靠花字体，靠克制字号阶梯 + 大呼吸。miniprogram 用 rpx：

| Token | rpx | 用途 |
|------|------|------|
| `--fs-display` | 64 | 单一页面大数字（综合评分） |
| `--fs-title` | 36 | 页面标题 |
| `--fs-h2` | 30 | 卡片标题 |
| `--fs-body` | 28 | 正文 |
| `--fs-caption` | 24 | 次要说明 |
| `--fs-label` | 22 | 标签/微小（letter-spacing 拉开造大写感） |

字重只用两档：`400` 正文、`600` 标题/数字。**不用 700/800**——重字重显模板感，克制 600 + 大字号才高级。

### 3.3 间距 / 圆角 / 阴影

| Token | 值 | 说明 |
|------|------|------|
| `--sp-1` ~ `--sp-6` | 8/16/24/32/48/64 | 4 的倍数阶梯 |
| `--radius-sm` | 12 | 标签/小按钮 |
| `--radius` | 20 | 卡片 |
| `--radius-lg` | 28 | 大容器/hero |
| `--shadow` | `0 2rpx 12rpx rgba(15,23,42,0.05)` | 卡片，极轻 |
| `--shadow-brand` | `0 4rpx 16rpx rgba(15,118,110,0.16)` | 主按钮/数据图，唯一带色阴影 |

**留白铁律**：页面横向 padding 统一 `--sp-4`(32rpx)，卡片间距 `--sp-3`(24rpx)，卡片内 padding `--sp-4`，section 间纵向 `--sp-5`(48rpx) 起步——比现状大近一倍，这是"模板感→高级感"最关键的一刀。

## 4. 组件清单（混合架构）

**边界规则**：单元素、无内部状态、纯排版 → 工具类；含 canvas / 多 props / 跨 3+ 页复用 → 自定义组件。

### 4.1 工具类（`app.wxss`，沿用并真正用起 token）

| 类别 | 类 | 说明 |
|------|------|------|
| 布局 | `.flex-row` `.flex-col` `.flex-between` `.flex-center` `.grid-2` | 与现有同名保留 |
| 间距 | `.p-sm/md/lg` `.mt-md` `.gap-md` | 改为引用 `--sp-*` 变量 |
| 文字 | `.t-display` `.t-title` `.t-h2` `.t-body` `.t-caption` `.t-label` `.t-muted` | 引用 `--fs-*`，标签类加 letter-spacing |
| 卡片 | `.card` `.card-flat` | `.card` 白底+轻影+20rpx；`.card-flat` 无影纯边框（留白场景） |

### 4.2 自定义组件（`components/`，4 个）

| 组件 | props | 说明 |
|------|------|------|
| `kb-radar` | `data[8]` `mode(result\|compare)` `prevData[8]` | canvas 2d 绘制，无三方库。result=色块填充+问题点高亮；compare=双层叠加。入场动画：多边形从中心展开 400ms |
| `kb-score` | `value` `label` `delta` `variant(ring\|plain)` | 大号 display 数字 + delta。首页摘要用 plain，结果页用 ring |
| `kb-quota` | `items[{label,remaining,total}]` | 横向配额条组，teal 进度条，超量变橙 |
| `kb-empty` | `icon` `text` `cta` | 统一空态，替换现状文字字符 `○` 占位 |

## 5. 雷达图组件规格（招牌数据图）

8 维度固定顺序：头前伸 / 圆肩 / 骨盆前倾 / 膝超伸 / 脊柱侧弯 / 高低肩 / XO 型腿 / 核心稳定。
轴线角度：`-90° + i × 45°`（i=0..7），8 轴。网格为同心圆（4 圈：25/50/75/100%），非八边形——更干净更极简。
数据多边形 8 顶点：`center + r × (value/100) × (cos,sin)`。

### 5.1 变体 2 · result 模式（分析结果页 + 首页摘要）

- 网格：`--line-soft` 1px 同心圆 + 极淡轴线
- 数据多边形：`--brand` 12% 透明填充 + 2px `--brand` 描边 + 顶点 `--brand` 圆点（r=3.5）
- **问题点高亮**：最低分维度的顶点加 `--accent-warn` 2px 描边圆环（r=6.5）——全图唯一橙色，抓眼
- 标签外置：`--text-3` 小字，按象限 text-anchor 对齐

### 5.2 变体 3 · compare 模式（历史对比页）

- 上次数据：`#cbd5e1` 1.5px 虚线描边（dasharray 4 3），无填充
- 本次数据：`--brand` 12% 填充 + 2px 描边 + 顶点圆点
- 图例：本次（teal 实线块）/ 上次（灰虚线）
- 讲 before/after 进步故事

### 5.3 实现要点

- miniprogram：原生 canvas 2d API 绘制，**不引第三方图表库**，包体零增量
- 入场动画：多边形从中心（r=0）展开到目标 r，400ms ease-out
- 尺寸：结果页 280×280，首页摘要/对比页可缩放
- Flutter 端：参照本规范重构现有 `radar_chart.dart`（修掉彩虹图例 bug，统一为单色 teal + 问题点高亮）
- Web 端：SVG 实现同规范

## 6. 图标系统

统一一套线性 SVG，三端共用同一组源文件。

### 6.1 统一规则

- 描边 `1.8`，`stroke-linecap=round` `stroke-linejoin=round`
- viewBox `24×24`，无填充色块，纯线性
- inactive：`--text-2`(`#475569`)；active：`--brand`(`#0f766e`)
- 三端载体：miniprogram 内联 svg（wxml 支持）；Flutter 同源 svg 转 `flutter_svg`；Web 直接用 svg

### 6.2 图标分组

| 分组 | 数量 | 用途 |
|------|------|------|
| Tab 图标 | 5 | 首页/分析/训练/对话/动作库 |
| 8 维体态图标 | 8 | 雷达图例/结果页维度卡/历史列表，按"人体部位+问题方向"语义 |
| 状态语义图标 | 5 | 达标✓ / 问题点⚠(橙) / 上升↗ / 下降↘(灰) / 信息ℹ |
| 动作模式图标 | 8 | 推/拉/蹲/髋铰链/核心/拉伸/活动度/呼吸 |

### 6.3 动作库图标策略（可扩展核心）

**坚决不为单个动作做图标**——按"一动作一图标"做，28 个还行，扩到 200+ 必崩（每个新动作找设计、风格漂移、维护地狱）。

改为**动作模式图标集 + 肌群色标**：

- **图标层**（8 个动作模式，固定不变）：任何动作都属于 8 种模式之一，新动作只是现有模式变体。DB 加 `pattern` 枚举（8 选 1）字段，UI 据 `pattern` 渲染对应模式图标。
- **色标层**（6 色数据色板，固定不变）：据 `targetMuscle`（现有 `Suggestion` model 已有该字段）映射肌群色点，贴在模式图标旁。
- **动作条目**：无限扩展，DB 填 `pattern` + `targetMuscle` 即自动获得视觉，**零图标设计成本**。

模式与覆盖：
- 推 Push（俯卧撑/墙推）｜拉 Pull（划船/后拉）｜蹲 Squat（深蹲/静蹲）｜髋铰链 Hinge（硬拉/臀桥）｜核心 Core（平板/死虫）｜拉伸 Stretch（静态拉伸）｜活动度 Mobility（猫牛/胸椎旋转）｜呼吸 Breathing（腹式呼吸）

**不选其他方案的理由**：每动作配照片/插画（风格难统一/版权/体积膨胀）；每动作首字母 badge（无信息量）；每动作单独 SVG（维护地狱，200+ 必崩）。
